## 📖 项目简介

ComfyNexus 是一个专为 ComfyUI 设计的现代化管理工具，提供直观的可视化界面和强大的管理功能。无论您是 AI 创作新手还是资深用户，ComfyNexus 都能帮助您更轻松地管理 ComfyUI 环境、插件、模型和工作流。

### ✨ 核心亮点

- 🎯 **一站式管理** - 环境、版本、插件、模型，一个应用全搞定
- ⚡ **智能优化** - 内置多种性能预设，极客模式支持自定义启动参数
- 🛡️ **智能保护** - 自动检测进程冲突，防止端口占用和启动失败
- 🎨 **现代界面** - 精美的深色主题，流畅的交互体验
- 🌍 **多语言支持** - 中文/英文界面，自动适配系统语言
- 📦 **配置同步** - 支持预设导入导出，轻松迁移和备份

### 🎬 快速预览

ComfyNexus 提供了完整的 ComfyUI 管理解决方案，从环境配置到模型管理，从插件安装到工作流编排，让您专注于创作本身。

### 15. 🛡️ 进程冲突检测

智能检测和处理 ComfyUI 进程冲突，确保顺利启动。

**自动检测**
- 启动前自动扫描系统进程
- 精准识别 ComfyUI 进程
- 自动排除当前管理的进程
- 智能端口冲突检测

**冲突信息**
- 进程 ID (PID)
- 监听端口（冲突端口特别标注）
- 工作目录
- 创建时间

**处理方式**
1. **结束不受管理的进程**（推荐）

   - 终止后自动继续启动

2. **继续启动**
   - 忽略冲突警告，强制启动
   - 适用于确认无影响的情况
   - 可能因端口冲突导致启动失败

3. **取消**
   - 关闭对话框，不执行操作
   - 适用于需要手动处理的情况
   - 支持 ESC 键快速关闭

**智能识别**
- 避免误识别开发工具进程
- 支持多种端口参数格式
- 检测 ComfyUI 特有目录结构
- 权限不足自动提示

**自动清理**
- 应用关闭时自动检测运行中的进程
- 自动终止进程树（包括子进程）
- 防止遗留僵尸进程

**使用场景**

## 🎯 使用场景

### 场景 1：新手入门

**问题**：刚接触 ComfyUI，不知道如何配置和启动。

**解决方案**：
1. 打开 ComfyNexus，进入"环境设置"
2. 点击"扫描环境"，自动识别已安装的 ComfyUI
3. 选择一个环境，应用内置的"Flux 标准"预设
4. 返回首页，点击"启动 ComfyUI"
5. 进入"工作台"，开始创作

### 场景 2：多环境管理

**问题**：需要在不同的 ComfyUI 版本之间切换测试。

**解决方案**：
1. 在"环境设置"中添加多个环境
2. 为每个环境设置不同的别名和端口
3. 在顶部导航栏快速切换环境
4. 每个环境的配置独立保存，互不影响

### 场景 3：性能优化

**问题**：显卡性能不足，生成图片速度慢或内存不足。

**解决方案**：
1. 进入"环境设置" → "加速配置"
2. **普通模式**：根据显卡性能选择合适的预设
   - 高端显卡：选择"旗舰画质"
   - 中端显卡：选择"Flux 标准"
   - 低端显卡：选择"老显卡救星"
3. **极客模式**：高级用户可以直接编辑启动参数
   - 切换到极客模式
   - 在多行编辑器中输入自定义参数
   - 支持注释和空行
   - 自动验证参数格式
4. 保存配置，重启 ComfyUI 生效

### 场景 4：版本管理

**问题**：想尝试最新的开发版功能，但担心不稳定。

**解决方案**：
1. 进入"核心管理"
2. 切换到"开发版"标签
3. 浏览最新的提交记录
4. 选择想要的版本，点击"切换版本"
5. 如果遇到问题，随时切换回稳定版

### 场景 5：进程冲突

**问题**：启动 ComfyUI 时提示端口被占用。

**解决方案**：
1. ComfyNexus 自动检测到冲突进程
2. 弹出冲突对话框，显示冲突进程详情
3. 点击"结束不受管理的进程"
4. 自动清理冲突进程并继续启动
5. 无需手动打开任务管理器

## 💡 常见问题

### 安装和启动

**Q: ComfyNexus 支持哪些操作系统？**  
A: 目前主要支持 Windows 7+，基于 pywebview 6.x 和 React 19。macOS 和 Linux 支持正在开发中。

**Q: 需要安装 Python 吗？**  
A: 是的，需要 Python 3.12 或更高版本。ComfyNexus 会使用系统的 Python 环境。

**Q: 如何更新 ComfyNexus？**  
A: 访问 GitHub 下载最新版本，解压后运行即可。配置文件会自动迁移。

**Q: 窗口大小如何调整？**  
A: 在"系统设置" → "通用设置"中选择预设窗口大小，或直接拖动窗口边缘调整，设置会实时保存。

### 环境管理

**Q: 如何添加新环境？**  
A: 在顶部导航栏点击"+"按钮，选择 ComfyUI 目录即可。也可以使用"扫描环境"自动识别。

**Q: 环境切换后配置会丢失吗？**  
A: 不会。每个环境的配置独立保存，互不影响。配置文件有安全保护机制，防止意外丢失。

**Q: 可以同时运行多个环境吗？**  
A: 可以，但需要为每个环境配置不同的端口，避免端口冲突。

**Q: 如何删除环境？**  
A: 在顶部导航栏的环境下拉菜单中选择"删除环境"，确认后即可删除。注意：只删除配置文件，不删除 ComfyUI 目录。

### 进程冲突检测

**Q: 为什么会检测到冲突进程？**  
A: 可能是上次启动的 ComfyUI 未正常关闭，或者您手动启动了其他 ComfyUI 实例。

**Q: 结束进程会丢失数据吗？**  
A: 不会。结束的是遗留的僵尸进程或外部进程，不会影响已保存的工作流和配置。

**Q: 提示权限不足怎么办？**  
A: 以管理员身份运行 ComfyNexus，或手动在任务管理器中结束冲突进程。

**Q: 如何同时运行多个 ComfyUI 实例？**  
A: 为不同环境配置不同的端口号，避免端口冲突。

### 性能优化

**Q: 预设方案如何选择？**  
A: 根据您的显卡性能选择：
- 高端显卡（RTX 4090/4080）：选择"旗舰画质"
- 中端显卡（RTX 3060/4060）：选择"Flux 标准"
- 低端显卡（GTX 1660/2060）：选择"老显卡救星"

**Q: 普通模式和极客模式有什么区别？**  
A: 
- 普通模式：通过图形界面选择预设和调整参数，适合大多数用户
- 极客模式：直接编辑启动参数，适合高级用户和需要精细控制的场景

**Q: 极客模式如何使用？**  
A: 在"环境设置" → "加速配置"中切换到极客模式，在多行编辑器中输入启动参数，支持注释和空行。

**Q: 自定义预设如何创建？**  
A: 
- 普通模式：调整参数后点击"保存为预设"
- 极客模式：编辑参数后点击"保存为新预设"

**Q: 预设可以导出分享吗？**  
A: 可以，在预设管理中选择"导出预设"，生成配置文件分享给其他用户。

### 版本管理

**Q: 开发版和稳定版有什么区别？**  
A: 开发版包含最新功能但可能不稳定，稳定版经过测试更可靠。建议日常使用稳定版。首次切换到开发版会有警告提示，可以选择"不再提醒"。

**Q: 切换版本会影响已有的工作流吗？**  
A: 一般不会，但建议在切换前备份重要的工作流文件。

**Q: 如何回退到之前的版本？**  
A: 在"版本管理"中选择想要的版本，点击"切换版本"即可。支持无限滚动加载历史版本。

**Q: 切换环境后版本信息不更新怎么办？**  
A: 版本管理页面会自动检测环境切换并刷新版本信息，无需手动操作。
